HOW EVERYTHING WORKS

Quick start

  1. Load a database from Library, Open .emdb, CSV folder, or the bundled default DB.
  2. Choose a table tab: Players, Teams, Staff, Sponsors, or Tournaments.
  3. Use grid cards for visual editing, or list view for spreadsheet-style scanning.
  4. Select a row or card, then press the edit icon to change values and assets.
  5. Save as .emdb or export CSVs when you are done.

Databases

Use Library for bundled databases, Load Default DB for the NoScope default, Open .emdb for a local package, or Open CSV folder for loose table files.

Save as .emdb creates a game-ready database package. Export CSVs creates a ZIP of the editable table files.

Cards And Lists

Grid view shows visual cards. Player, staff, sponsor, and team cards can flip after hovering to reveal extra information.

List view is for scanning. Some tables have custom view presets, such as player attribute groups, staff role attributes, and team rosters.

Sorting And Search

The search box filters as you type. Sort controls change depending on the active table.

Players sort by rating leaderboard, name, or Team ERS. Teams sort by ERS or name. Sponsors sort by tier or name. Staff sort by role-aware rating, name, or Team ERS.

Editing

The editor separates identity, skills, details, and team roster tools where applicable. Calculated ratings are refreshed automatically when relevant attributes are saved.

Use Add to create a row, Remove to delete selected rows, and Deselect all to clear multi-selection.

Teams And Rosters

Team editor roster view lets you drag players to reorder main squad and bench. The first five assigned players are treated as main squad; remaining players are bench.

Team list roster view and flipped team cards use the same roster matching logic.

Images And Assets

Images are matched by stable identifiers such as internal ID, nickname, name, or team ID. Use the asset editor inside an entry to choose or remove a local image override.

The Assets button opens the CustomAssets Google Drive folder for shared image files.

Filters And Compare

Filter opens table-specific filters for narrowing the current table. Compare appears for Players and lets you inspect two player profiles side by side.

Localhost Note

Some browser security rules block bundled database loading from a direct file path. If Library cannot read databases, open the app through localhost.
