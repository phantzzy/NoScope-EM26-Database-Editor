HOW EVERYTHING WORKS

Start here

Four steps from database to finished file

  1. LoadOpen your `.emdb`, choose a bundled Library database, load the default, or open a CSV folder.
  2. ChooseOpen Players, Sponsors, Staff, Teams, or Tournaments and switch between card and list views.
  3. EditSearch, filter, select an entry, and use its pencil button. Add, remove, compare, or mass edit where available.
  4. Check and saveRun Validate, review any findings, then save the game-ready `.emdb` package.
In this guide Databases and saving Finding and selecting entries Editing players and records Teams and rosters Merge, mass edit, and validation Images and appearance
01

Databases And Saving

Load, combine, check, and export database packages.

Open a database

Open .emdb loads a local package. Library offers bundled databases, while Load Default DB opens the NoScope default. CSV folders remain available for loose-table work.

Save and export

Save as .emdb builds the encrypted game-ready package. Export CSVs creates a ZIP of the editable table files. Unsaved-change prompts appear before replacing the current database.

Merge Beta

The loaded database is always Database A. Choose Database B, select tables and player field groups, review every proposed addition or transfer, then apply only the approved changes.

Undo a merge

The last applied merge can be undone until another edit or save replaces that undo state. Merging never deletes entries from Database A automatically.

02

Finding And Selecting Entries

Move quickly through large databases without losing context.

Cards and lists

Card view is visual and includes hover-flip details. List view is designed for scanning and sorting, with focused presets for player attributes, staff roles, and team rosters.

Search, sort, and filter

Search updates as you type. Sort options change with the active table, while Filter exposes table-specific fields such as player gender, FACEIT origin, content creator status, teams, and attributes.

Select records

Click a card or list row to select it. Select multiple records for removal or player mass editing. Deselect all clears a multi-selection.

Compare players

Select one or two players before opening Compare. Two selected players are loaded directly into the comparison; selecting more than two shows a limit warning.

03

Editing Players And Records

Use each entry's focused editor instead of modifying raw table data.

Create and edit

Use Add for a new entry or the pencil icon for an existing one. Player, staff, and team editors separate identity, skills, details, and roster tools into focused pages.

Ratings and potential

Player and staff ratings recalculate from relevant attributes. Enter player Potential from 1-20, or leave it blank to calculate and write an automatic value when saving. Potential is displayed as five stars without changing the current Rating.

Unknown birthdays

Clear an existing birthdate before randomizing. Enter a known age or let NoScope generate an age from 16-26. Saving a blank player birthday defaults it to age 18.

Player badges

FACEIT-origin and NoScope content creator states appear as badges on player cards and lists. Both can also be used as player filters.

04

Teams And Rosters

Build the main squad and bench from inside the team editor.

Add players

Search the database from the Roster page and add matching players. Create Player opens the regular player editor and returns the new player to the roster workflow.

Main squad and bench

Drag players to reorder them or drop them directly into the Main squad or Bench area. The selected destination is saved even when fewer than five main players exist.

Roster state

Team membership is written back to player Team, Team ID, and Roster Position fields when those columns exist. Team cards and roster list view use the same saved state.

Tournament locations

Tournaments use eligible country and city combinations. The editor displays a fixed map preview for the selected location and rejects unsupported pairs.

05

Mass Editing And Validation

Use the bulk tools deliberately and review their output.

Mass Edit Beta

Select at least two players, then choose fixed values or random ranges for the fields you want to change. Unchecked fields remain untouched and ratings recalculate after applying.

Quick ranges

Tier presets assign common attribute ranges. Section controls select or clear related fields, while Reset restores the standard 1-20 ranges before applying.

Validate

The standalone Validate button checks duplicate identifiers and names plus missing important fields across all five database tables. Clean checks stay out of the way.

Pre-save review

Validation runs automatically before `.emdb` saving. Findings are grouped by table with entry samples. Return to the editor to fix them or choose Save anyway for intentional legacy data.

06

Images And Appearance

Control entry images and switch the editor theme.

Automatic image matching

Bundled images are matched using stable IDs, nicknames, names, and team identifiers. Use consistent entry names and IDs when adding records so their available images resolve correctly.

Local image overrides

Open an entry with its pencil button and use the image area to choose or remove a local override. The preview shows which image will be associated with that entry in NoScope.

Assets library

Open Assets from the header to access the shared CustomAssets collection for players, staff, sponsors, teams, and tournaments.

Light and dark mode

Dark mode is the startup default. Use the moon or sun button beside Assets to switch the current session between dark and light themes.
